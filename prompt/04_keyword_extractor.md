# Prompt 4：文档关键词自动提取（core/keyword_extractor.py）

**用途**：文档入库时自动提取部门归属和关键技术词，追加到 BM25 索引文本，解决文档原文词汇与用户查询词汇不匹配的问题（词汇鸿沟）。

**触发时机**：`POST /v1/documents` 入库时；以及服务启动时生成 index.json

**模型**：DeepSeek deepseek-chat

---

```
分析以下 On-Call SOP 文档，提取关键信息。
只输出 JSON，不要任何解释或 markdown：
{
  "department": "负责团队（从以下选一个：后端服务、SRE、数据库DBA、前端、安全团队、数据平台、移动端、AI算法、QA、网络CDN）",
  "keywords": ["关键技术术语1", "术语2"]
}

keywords 要求：最多 15 个，选对这份文档最有区分度的技术词，用于 BM25 检索匹配。

文档标题：{title}
文档内容（前 600 字）：{text}
```

---

**设计背景**：

sop-001（后端）原文从不出现"服务器"二字（使用"服务"、"Pod"等词），导致用户搜"服务器挂了"时该文档 BM25 得分极低。

解决思路演变：
1. **临时方案**：手动往10个 HTML 文件注入 `<div class="kw-index">` 关键词 div
2. **正式方案**：LLM 自动提取，追加到索引文本，HTML 文件本身不修改，支持任意数量文档

**实测输出（sop-001）**：
```json
{
  "department": "后端服务",
  "keywords": ["OOM", "内存溢出", "服务超时", "降级策略", "故障分级", "服务器崩溃", "服务宕机", "P99", "QPS", "可用率"]
}
```
